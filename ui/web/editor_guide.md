# VelDB Editor Operations Handbook

Welcome to the newly integrated query operations interface for VelDB! Use this toolset to rapidly design, verify, and enact deep structural changes into your data securely leveraging AI components seamlessly.

## ✨ 1. Leveraging the AI Natural Language Core 
You don't need to learn SQL perfectly to retrieve accurate data!

**How to use:**
1. Click straight into the `AI Query Assistant` interface bar.
2. Structure your goal using basic plain English formatting. 
   - *Example: "Select all users joined after January 1st ordered by revenue"*
3. Click **"Generate SQL"** (or hit Enter).
4. The system automatically communicates externally, generating a strictly formatted query. 

**Note on Safety:** To ensure strict database safety, the AI *never* auto-runs queries! It instantly populates your primary SQL block giving you perfect control to inspect its assumptions prior to executing variables blindly.

## ⚡ 2. Utilizing The Smart SQL Block 
Our main execution layer offers automatic syntax completions eliminating the need for strict memorization vectors.

**Using Autocomplete:**
1. Start typing normally (`SEL...` or `use...`).
2. A lightweight option window will automatically pull up suggesting exact keywords, table markers, and nested column structures.
3. Keep typing to filter the context box manually, or simply utilize standard your `<Up>` and `<Down>` keys navigating the focused options!
4. Hit `<Enter>` or `<Tab>` instantly inserting the block, avoiding strict structural spelling typos entirely!

## 🚀 3. Execution
When you are fully satisfied with your query schema and its parameters, simply locate the large **"Run Query"** button highlighted directly beneath your text block. 
*(This button remains greyed out ensuring empty payloads are never sent to the network reducing spam allocations perfectly).*
